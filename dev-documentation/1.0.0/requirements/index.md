# Requirements

This section lists the functional requirements for CEON that are extracted from the overview Onto-DESIDE project requirements. 
These requirements serve as the basis for the [Competency Questions](../competency-questions/index.md).

## How Requirements Are Organized

Each requirement is represented in two perspectives. One is `Circular Enabler Category (CEC)`, the other is `Circular Enablers (sotry)`, which has a unique identifier (`CE<NUMBER>`).

| Field | Description |
|---|---|
| **ID** | Unique identifier, e.g. `CE1` |
| **Description** | What the ontology must be able to represent |
| **CQs** | Which competency questions this requirement motivates |
| **Reasoning Requirements** | what corresponding reasoning requirements are |
